# Source Code Base: rustzk

## Directory Structure

```
rustzk/
├── .agent/             # Agent configuration and rules.
├── .github/            # GitHub Actions (CI/CD workflows).
├── docs/               # Documentation files (generated by this process).
├── examples/           # Usage examples for the library.
│   ├── check_device_info.rs  # Example: Connect and get basic info.
│   ├── debug_log_count.rs    # Example: Debug log retrieval issues.
│   └── main.rs               # Example: General usage.
├── plans/              # Planning documents.
├── pyzk_ref/           # Reference Python implementation (pyzk).
├── reports/            # Generated reports.
├── scripts/            # Utility scripts (e.g., connect.sh).
├── src/                # Source code.
│   ├── lib.rs          # Main library file (lib root).
│   ├── constants.rs    # Protocol constants (CMD_*, EF_*, etc.).
│   ├── models.rs       # Data structures (User, Attendance, etc.).
│   ├── protocol.rs     # Protocol implementation (ZKPacket, TCPWrapper).
│   ├── transport.rs    # Transport layer abstraction (TCP/UDP).
│   └── error.rs        # Error definitions.
├── tests/              # Integration tests.
│   ├── client_tests.rs # Comprehensive client tests (mock server).
│   └── ...
├── Cargo.toml          # Package configuration and dependencies.
└── README.md           # Project README.
```

## Key Files
- **`src/lib.rs`:** The core of the library. Contains the `ZK` struct and its implementation.
- **`src/protocol.rs`:** Handles packet encoding and decoding. Crucial for understanding the binary format.
- **`src/models.rs`:** Defines the data structures returned by the API.
- **`tests/client_tests.rs`:** Contains detailed integration tests using a mock ZK server. Good for understanding expected behavior and edge cases.

## Entry Points
- **Library User:** Import `rustzk::ZK` and `rustzk::ZKProtocol`.
- **Contributor:** Start with `src/lib.rs` to understand the high-level logic, then dive into `src/protocol.rs` for low-level details.

## Modules
- **`constants`:** Centralized location for all protocol magic numbers.
- **`models`:** Domain entities.
- **`protocol`:** Serialization/Deserialization logic.
- **`transport`:** Networking abstraction.
- **`error`:** Error types.
